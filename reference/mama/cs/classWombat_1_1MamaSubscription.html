Allocate memory for a new subscription.

§ activate()

void Wombat.MamaSubscription.activate ( )

Activate a subscription that has been set up by calling the setup function.

Subscription creation actually occurs on the throttle queue. An individual subscription cannot be assumed to be fully created until its onCreate callback has been successfully invoked. The subscription rate can be governed via the MamaTransport.setOutboundThrottle function. Any subscription properties should be set prior to calling this function.

§ setPreInitialCacheSize()

void Wombat.MamaSubscription.setPreInitialCacheSize ( int  cacheSize)

Set the number of messages to cache for each symbol before the initial value arrives.

This allows the subscription to recover when the initial value arrives late (after a subsequent trade or quote already arrived). For group subscription, a separate cache is used for each group member. The default is 10.

Parameters
cacheSizeThe cache size to set.
